# 瞬聊优化清单

## 目标
- 将当前项目从“可联调测试版”推进到“可交付上线版”
- 优先处理：核心链路稳定性、UI 统一性、环境隔离、数据持久化

## P0：必须优先处理

### 1. 聊天核心链路稳定性
- [ ] 匹配成功后建会话、发消息、收消息全链路回归
- [ ] 发送失败时给出明确可理解的提示与重试入口
- [ ] 进入旧会话时自动恢复真实会话 ID 的逻辑继续验证
- [ ] 图片消息、已读状态、撤回/闪图场景补全回归用例

### 2. 环境与演示逻辑分层
- [ ] 明确 `demo/dev/staging/prod` 环境配置
- [ ] 将固定候选人池限制在 demo/dev 环境
- [ ] 将测试验证码、演示账号逻辑限制在测试环境
- [ ] 补一份环境变量说明文档

### 3. 数据持久化准备
- [ ] 将用户、设置、聊天会话、消息、匹配配额迁移到 PostgreSQL
- [ ] 将验证码、会话态、运行时缓存迁移到 Redis
- [ ] 为媒体资源准备对象存储/本地持久卷方案

## P1：高优先级体验优化

### 4. 匹配页体验优化
- [x] 强化匹配按钮为单一主操作
- [ ] 匹配中的加载态更轻、更稳定
- [x] 匹配成功页增强“命中感”和行动引导
- [x] 增加首条消息推荐文案，降低用户开场成本
- [x] 将“位置未开启”提示改成更产品化表达

### 5. 聊天页体验优化
- [x] 输入区层级更清晰，发送按钮更明显
- [x] 空状态页从“能用”升级为“鼓励开聊”的产品文案
- [x] 失败消息、取关限制、图片解锁提示统一视觉样式
- [ ] 聊天内系统提示减少打扰感，提升一致性

### 6. 首页与导航优化
- [x] 首页聚焦 3 个主动作：匹配、消息、我的
- [x] 底部导航统一选中/未选中态
- [x] 消息未读、通知角标层级统一
- [x] 页面切换和返回路径继续简化

### 7. 设置页信息架构优化
- [x] 将设置拆为：账号与安全 / 隐私 / 通知 / 关于
- [x] 减少高密度信息堆叠
- [ ] 保留核心项，弱化不常用项
- [x] 黑名单、手机号、UID 操作文案更清晰

## P2：UI 设计统一

### 8. 夜间主题系统化
- [x] 继续固定夜间主题，彻底去除日间设计分支
- [ ] 统一卡片、弹窗、输入框、底栏的夜间材质感
- [ ] 统一高亮色、错误色、弱提示色使用规范
- [ ] 统一标题、正文、辅助文字、按钮的层级

### 9. 设计规范沉淀
- [ ] 沉淀颜色、间距、圆角、描边、阴影标准
- [ ] 将重复视觉写法收敛为 UI token / 通用组件
- [ ] 收敛零散的文案与提示样式

## P3：工程与上线准备

### 10. 代码结构优化
- [ ] 继续拆分 `ChatProvider`，降低单文件复杂度
- [ ] 将聊天发送、实时同步、会话升级逻辑拆成更清晰层次
- [ ] 将匹配、设置、通知等状态逻辑继续模块化

### 11. 测试与质量
- [ ] 补充登录、匹配、建会话、发消息、已读的自动化测试
- [ ] 输出多轮测试报告与问题单
- [ ] 建立每次发版前的最小回归清单

### 12. 发布交付
- [ ] 正式服务器、数据库、Nginx、HTTPS 部署
- [ ] 正式签名、AAB、应用图标、启动图、隐私协议补齐
- [ ] 接入正式推送、监控、日志上报
- [ ] 形成上线部署文档与运维文档

## 页面级优化建议

### 首页
- 保留“进入匹配”作为最强主路径
- 弱化非核心信息噪音
- 提高首次用户理解成本的可视化引导

### 匹配页
- 让用户一眼知道“点击开始匹配”
- 匹配成功后优先引导“立即打招呼”
- 快捷开场白做成更轻的卡片式选择

### 聊天页
- 优化空状态文案
- 优化输入栏、发送按钮、图片入口层次
- 提升失败消息、限制消息、系统消息的一致性

### 我的 / 设置页
- 降低信息密度
- 提升安全、隐私、通知等分组清晰度
- 让操作反馈更明确

## 建议开发顺序
1. 聊天链路稳定性回归
2. 匹配页体验优化
3. 聊天页视觉与提示统一
4. 设置页信息架构收敛
5. 环境隔离
6. PostgreSQL / Redis 接入
7. 自动化测试补齐
8. 部署上线准备

## 下一批建议直接开始做
- 第一批：匹配页体验优化 + 首条消息引导
- 第二批：聊天页空状态、失败提示、输入区统一
- 第三批：设置页信息架构和文案收敛
- 第四批：后端环境隔离与持久化改造
