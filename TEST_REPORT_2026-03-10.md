## 4. 核心业务场景审查结果

### 4.1 登录场景

检查项：

- 验证码发送
- 验证码登录
- 登录态持久化
- 切换账号会话隔离

发现：

- 之前存在“接口失败时自动走本地伪登录”的逻辑
- 这会导致后端异常、服务器未连接时，客户端仍然可能显示登录成功

风险等级：高

处理结果：已修复

### 4.2 匹配场景

检查项：

- 发起匹配
- 配额扣减
- 远端匹配结果消费
- 无服务时失败表现

发现：

- 之前存在“远端匹配失败时回退到本地 mock 用户”的逻辑
- 这会导致用户误以为真实匹配成功，但实际上并未接入服务器

风险等级：高

处理结果：已修复（发布模式下不再伪造匹配对象）

### 4.3 聊天场景

检查项：

- 文本发送
- 图片发送
- 重发失败消息
- 无会话状态下的发送行为

发现：

- 之前在无有效会话时，文本/图片消息可能走本地模拟发送成功路径
- 这会造成“消息看起来发出去了，但服务端根本没有收到”的错觉

风险等级：高

处理结果：已修复（发布模式下改为失败，不再伪成功）

### 4.4 设置与法务场景

检查项：

- 关于页
- 隐私政策
- 用户协议

结果：

- 已从占位提示升级为正式页面
- 路由、设置入口、登录页文案复用均已接通

风险等级：低

处理结果：已完成

## 5. 本轮已修复问题

### 5.1 发布模式下禁止伪登录

文件：

- `flutter-app/lib/services/auth_service.dart`
- `flutter-app/lib/config/app_env.dart`

修复内容：

- 新增 `SUNLIAO_RELEASE_BUILD` 环境开关
- 发布模式下，验证码发送失败不再自动生成本地请求 ID
- 发布模式下，登录失败不再自动回退到本地假账号
- 登录失败时保留真实错误语义，不把服务异常误报成验证码错误

### 5.2 发布模式下禁止伪匹配

文件：

- `flutter-app/lib/providers/match_provider.dart`
- `flutter-app/lib/config/app_env.dart`

修复内容：

- 发布模式下，远端匹配失败时不再生成 mock 用户
- 避免区域测试和正式环境出现“假匹配成功”

### 5.3 发布模式下禁止离线消息伪成功

文件：

- `flutter-app/lib/providers/chat_provider.dart`
- `flutter-app/lib/config/app_env.dart`

修复内容：

- 发布模式下，无会话状态文本发送直接标记失败
- 发布模式下，重发失败消息不再本地模拟成功
- 发布模式下，图片发送失败直接标记失败

- Flutter 全量自动化测试与完整构建结果本轮未在当前环境中成功拿到
- 推送、监控、正式短信、正式签名、正式服务器等仍未全部闭环

## 7. 剩余风险与建议

### 7.1 高优先级

- 在你的机器上完成一次完整 APK 构建
- 真机联调登录 / 匹配 / 聊天 / 图片发送
- 接入真实后端后再做一次完整业务回归

### 7.2 中优先级

- 为登录 fallback、匹配 fallback、聊天失败处理补单元测试
- 为匹配失败增加更明确的 UI 提示
- 为资料/设置保存失败增加前端错误反馈

## 8. 本轮结论摘要

本轮最重要的修复，不是 UI 微调，而是把“假成功、假登录、假匹配、假发送”这类高风险逻辑从发布路径里收掉。这样后续你拿 APK 去测试时，看到的结果会更接近真实服务器行为，不会再被本地 mock 误导。
