# 在线状态与位置权限联动系统

## 📋 需求说明

### 核心逻辑
1. **匹配策略**：优先匹配在线用户，再匹配离线用户
2. **位置显示**：离线用户=没有位置权限，不显示位置信息
3. **实时更新**：用户上线后，如果开启位置，实时更新状态
4. **视觉吸引**：在线用户在消息页有明显的视觉吸引

### 好友与陌生人共存
- **好友消息**：无倒计时，有蓝色"好友"标签
- **陌生人消息**：有倒计时，显示亲密度，在线用户有绿点和"在线"标签

## 💻 技术实现

### 1. User模型扩展

```dart
class User {
  final String id;
  final String nickname;
  final String? avatar;
  final String distance;
  final String status;
  final bool isOnline;              // 是否在线
  final bool hasLocationPermission; // 是否开启位置权限
  final DateTime? lastOnlineTime;   // 最后在线时间
}
```

### 2. 匹配策略

```dart
User _generateMockUser() {
  // 70%概率匹配到在线用户（优先在线）
  final isOnline = DateTime.now().millisecond % 10 < 7;
  
  // 在线用户80%有位置权限，离线用户0%有位置权限
  final hasLocationPermission = isOnline && (DateTime.now().second % 10 < 8);
  
  return User(
    distance: hasLocationPermission 
        ? '${distance}km'
        : '位置未知',  // 离线用户不显示位置
    isOnline: isOnline,
    hasLocationPermission: hasLocationPermission,
  );
}
```

## 🔄 实时更新机制

### 状态变化场景

#### 场景1：用户上线
```
离线状态 → 在线状态
- 头像出现绿点
- 显示"在线"标签
- 背景变为微妙渐变
- 如果有位置权限，显示距离
```

#### 场景2：用户下线
```
在线状态 → 离线状态
- 绿点消失
- "在线"标签消失
- 背景恢复普通
- 位置信息隐藏
```

#### 场景3：用户开启位置
```
在线无位置 → 在线有位置
- 距离信息从"位置未知"变为具体距离
- 匹配卡片显示位置图标
```
