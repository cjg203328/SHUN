# 位置权限逻辑说明

## 需求描述

位置权限应该是**会话级别的缓存**，而不是持久化存储：

1. **首次打开APP** → 第一次匹配时请求位置权限
2. **用户选择** → 可以选择"使用位置"或"暂不使用"
3. **本次会话记忆** → 只要不杀后台，本次会话内不再弹窗
4. **杀后台重启** → 下次打开APP，首次匹配时重新请求
5. **位置图标** → 如果用户授权，手机顶部状态栏显示位置图标

## 实现方案

### 1. 会话级别缓存（内存存储）

```dart
class PermissionManager {
  // 会话级别的位置权限缓存（内存中，不持久化）
  static bool? _sessionLocationPermission;
  
  // 检查本次会话是否已请求过
  static bool hasSessionLocationPermission() {
    return _sessionLocationPermission != null;
  }
  
  // 获取本次会话的权限状态
  static bool? getSessionLocationPermission() {
    return _sessionLocationPermission;
  }
  
  // 保存本次会话的权限状态
  static void setSessionLocationPermission(bool granted) {
    _sessionLocationPermission = granted;
  }
  
  // 清除会话缓存（杀后台时自动清除）
  static void clearSessionCache() {
    _sessionLocationPermission = null;
  }
}
```

### 2. 请求逻辑

```dart
static Future<bool> requestLocationPermission(BuildContext context) async {
  // 1. 检查会话缓存
  if (hasSessionLocationPermission()) {
    final cached = getSessionLocationPermission();
    if (cached == true) {
      return true;  // 本次会话已授权
    } else if (cached == false) {
      return false; // 本次会话已拒绝
    }
  }
  
  // 2. 首次请求，显示选择弹窗
  final shouldUse = await _showLocationSelectionDialog(context);
  
  if (!shouldUse) {
    setSessionLocationPermission(false);
    return false;
  }
  
  // 3. 请求系统权限
  final status = await Permission.location.status;
  
  if (status.isGranted) {
    setSessionLocationPermission(true);
    return true;
  }
  
  if (status.isDenied) {
    final result = await Permission.location.request();
    setSessionLocationPermission(result.isGranted);
    return result.isGranted;
  }
  
  // 4. 永久拒绝，引导去设置
  if (status.isPermanentlyDenied) {
    await _showGoToSettingsDialog(context);
    setSessionLocationPermission(false);
    return false;
  }
  
  return false;
}
```

## 使用场景

### 场景1：首次打开APP
```
用户打开APP → 点击"开始匹配" → 弹出位置权限选择
→ 用户选择"使用位置" → 请求系统权限 → 授权成功
→ 保存到会话缓存（_sessionLocationPermission = true）
→ 顶部状态栏显示位置图标 📍
```

### 场景2：本次会话内再次匹配
```
用户再次点击"开始匹配" → 检查会话缓存
→ 发现已授权 → 直接使用位置，不弹窗
→ 顶部状态栏继续显示位置图标 📍
```

### 场景3：杀后台重启
```
用户杀掉APP后台 → 会话缓存自动清空（_sessionLocationPermission = null）
→ 重新打开APP → 点击"开始匹配" → 再次弹出位置权限选择
→ 用户重新选择
```

### 场景4：用户拒绝位置
```
用户选择"暂不使用" → 保存到会话缓存（_sessionLocationPermission = false）
→ 本次会话内不再弹窗 → 使用随机匹配策略
→ 顶部状态栏无位置图标
```

## 与持久化存储的区别

| 特性 | 会话缓存（当前方案） | 持久化存储（旧方案） |
|------|---------------------|---------------------|
| 存储位置 | 内存（静态变量） | SharedPreferences |
| 生命周期 | APP进程存活期间 | 永久保存 |
| 杀后台 | 自动清空 | 保留 |
| 重启APP | 重新请求 | 不再请求 |
| 位置图标 | 授权时显示 | 授权时显示 |
| 用户体验 | 每次启动确认 | 一次授权永久有效 |

## 匹配策略

### 有位置权限
```dart
if (hasLocationPermission) {
  // 优先匹配附近的人（基于GPS定位）
  matchNearbyUsers();
}
```

### 无位置权限
```dart
if (!hasLocationPermission) {
  // 随机匹配全国用户
  matchRandomUsers();
}
```

## 位置图标显示

### Android
- 授权后，系统自动在状态栏显示位置图标 📍
- 图标由系统管理，APP无需额外处理
- 杀后台后图标自动消失

### iOS
- 授权后，状态栏显示位置箭头 ➤
- 图标由系统管理
- 杀后台后图标自动消失

## 优势

1. ✅ **符合用户预期**：杀后台=重新开始
2. ✅ **隐私保护**：不持久化位置偏好
3. ✅ **灵活控制**：每次启动可重新选择
4. ✅ **系统图标**：授权时自动显示位置图标
5. ✅ **简单实现**：无需数据库，内存缓存即可

## 注意事项

1. **会话缓存自动清空**：杀后台、重启APP、系统回收内存时自动清空
2. **不影响系统权限**：系统级别的位置权限仍然保留（用户在系统设置中授权后，下次不需要再授权系统权限，只需要在APP内选择是否使用）
3. **位置图标**：只有实际使用位置服务时才显示，不使用时自动消失
