## 一、产品概述

### 1.1 产品定位
瞬是一款极简、克制的24小时限时匿名社交应用，专注于真实而短暂的人际连接。

### 1.2 核心理念
- **限时性**: 24小时后对话自动消失，珍惜当下
- **匿名性**: 渐进式身份解锁，保护隐私
- **克制性**: 每日20次匹配机会，避免过度沉迷
- **纯粹性**: 极简设计，专注对话本身

### 1.3 目标用户
- **年龄**: 18-35岁
- **特征**: 喜欢深度交流，追求真实连接，注重隐私保护
- **场景**: 深夜失眠、情绪低落、想找人聊天、寻求陪伴

### 1.4 产品价值
- **用户价值**: 提供安全、真实、有温度的社交体验
- **社会价值**: 缓解孤独感，促进真实交流
- **商业价值**: 会员订阅、增值服务

## 三、功能详细设计

### 3.1 启动模块

#### 3.1.1 启动页
**功能描述**:  
应用启动时的品牌展示页面

**设计要求**:
- 纯黑背景（#000000）
- 光球呼吸动画（3秒循环）
- 品牌名"瞬"（56号超轻字重，16px字间距）
- 2.5秒后自动跳转登录页

**交互逻辑**:
```
启动APP → 显示启动页 → 光球呼吸动画 → 2.5秒后 → 跳转登录页
```

#### 3.1.2 登录注册
**功能描述**:  
用户通过手机号+验证码登录

**页面元素**:
- 品牌名"瞬"（顶部居中）
- 手机号输入框（限制11位数字）
- 验证码输入框（6位数字）
- 获取验证码按钮（60秒倒计时）
- 登录按钮
- 用户协议和隐私政策勾选

**交互逻辑**:
```
输入手机号 → 点击获取验证码 → 60秒倒计时 → 输入验证码 → 点击登录
  ↓
验证成功 → 跳转主页面
验证失败 → 显示错误提示（AppToast）
```

**验证规则**:
- 手机号必须为11位数字
- 验证码为6位数字
- Mock验证码：123456

### 3.2 匹配模块

#### 3.2.1 匹配页面
**功能描述**:  
随机匹配陌生人，每日20次机会

**页面布局**:
```
┌─────────────────────┐
│      品牌名"瞬"      │  ← 顶部标题
│                     │
│    情感文案区域      │  ← 中部文案
│                     │
│   ◉ 光球动画 ◉      │  ← 核心视觉
│                     │
│   今日剩余 20 次    │  ← 次数显示（大号数字）
│                     │
│   [开始匹配]按钮    │  ← 底部按钮
└─────────────────────┘
```

**状态设计**:
1. **初始状态**: 显示"开始匹配"按钮，显示剩余次数
2. **匹配中**: 按钮变为"取消"，光球呼吸动画加速
3. **匹配成功**: 弹出用户卡片，立即消耗1次
4. **次数用完**: 按钮变灰，显示"今日已用完"

**交互逻辑**:
```
点击"开始匹配" → 匹配中（2.5秒）
  ↓
  可以点"取消" → 不消耗次数 ✅
  ↓
匹配成功（弹出卡片） → 立即消耗1次 ✅
  ↓
  点"算了" → 关闭卡片，不再消耗
  点"发送招呼" → 创建聊天，不再消耗
```

#### 3.2.2 用户卡片
**功能描述**:  
匹配成功后弹出的用户信息卡片

**卡片内容**:
- 用户头像（模糊）
- 用户昵称
- 年龄、性别
- 距离（可选）
- 在线状态

**操作按钮**:
- "算了"按钮（关闭卡片）
- "打个招呼"按钮（进入招呼页）

#### 3.2.3 招呼语页面
**功能描述**:  
选择或自定义招呼语

**设计要求**:
- 快捷语和自定义输入二选一
- 选择快捷语时隐藏输入框
- 开始输入时隐藏快捷语
- 切换流畅自然

**快捷语列表**:
- "嗨，很高兴认识你"
- "你好呀，聊聊天吧"
- "Hi，在做什么呢"
- 自定义输入框

**交互逻辑**:
```
选择快捷语 → 隐藏输入框 → 点击发送
  或
点击输入框 → 隐藏快捷语 → 输入内容 → 点击发送
  ↓
创建聊天会话 → 跳转聊天页面
```

### 3.3 聊天模块

#### 3.3.1 聊天页面
**功能描述**:  
与匹配用户进行实时聊天

**页面结构**:
```
┌─────────────────────────────┐
│ ← [头像] 昵称 [更多]        │  ← 标题栏
│   15分钟解锁头像             │  ← 解锁提示
├─────────────────────────────┤
│                             │
│  [对方消息气泡]              │
│                             │
│           [我的消息气泡]     │
│                             │
│  [对方消息气泡]              │
│                             │
├─────────────────────────────┤
│ [+] [输入框...] [发送]      │  ← 输入栏
└─────────────────────────────┘
```

**标题栏显示**:
- 0-15分钟: "X分钟解锁头像"
- 15-30分钟: "X分钟解锁主页"
- 30分钟后: "可以添加好友"
- 好友状态: "好友"

**消息类型**:
- 文本消息
- 系统消息（解锁提示）
- 撤回消息（显示"已撤回"）

**消息操作**:
- 长按消息 → 弹出操作菜单
  - 复制（所有消息）
  - 撤回（2分钟内，仅自己的消息）

#### 3.3.2 渐进式解锁
**功能描述**:  
随着聊天时间增加，逐步解锁对方信息

**解锁时间线**:
```
0分钟: 匹配成功
  ↓
15分钟: 解锁头像（显示真实头像）
  ↓
30分钟: 解锁主页（可查看个人主页）+ 可添加好友
  ↓
24小时: 对话消失（除非成为好友）
```

**系统提示**:
- 15分钟时: "你们已经聊了15分钟，头像已解锁"
- 30分钟时: "你们已经聊了30分钟，可以添加好友了"
- 23小时时: "对话将在1小时后消失"

#### 3.3.3 更多菜单
**功能描述**:  
聊天页面右上角的更多操作

**菜单选项**:
- 查看个人主页（30分钟后解锁）
- 语音通话（好友或30分钟后）
- 添加好友（30分钟后）
- 举报用户
- 删除对话

### 3.4 好友模块

#### 3.4.1 好友系统流程
**功能描述**:  
从陌生人到好友的完整流程

**流程设计**:
```
陌生人匹配 → 开始聊天
  ↓
聊天15分钟 → 解锁头像
  ↓
聊天30分钟 → 解锁主页 + 可以添加好友
  ↓
点击"更多"→"添加好友"
  ↓
填写附言（可选）→ 发送请求
  ↓
对方收到通知（红点）
  ↓
对方接受 → 成为好友
  ↓
好友聊天（无时间限制，完全解锁）
```

#### 3.4.2 好友请求
**功能描述**:  
发送和接收好友请求

**发送请求**:
- 触发条件: 聊天30分钟后
- 附言输入: 可选，最多50字
- 发送后: 显示"已发送好友请求"

**接收请求**:
- 通知方式: 好友Tab右上角红点
- 请求列表: 显示所有待处理请求
- 请求信息: 头像、昵称、附言、时间

**操作按钮**:
- "接受"按钮 → 成为好友
- "拒绝"按钮 → 更新请求状态

#### 3.4.3 好友列表
**功能描述**:  
显示所有好友

**列表项内容**:
- 好友头像
- 好友昵称（或备注）
- 成为好友时间
- 最后聊天时间

**操作功能**:
- 点击 → 进入聊天
- 长按 → 弹出菜单
  - 设置备注
  - 查看主页
  - 删除好友（二次确认）

#### 3.4.4 好友聊天
**功能描述**:  
与好友的聊天无时间限制

**特点**:
- 无24小时限制
- 完全解锁状态
- 可以随时聊天
- 支持所有聊天功能

### 3.7 权限管理

#### 3.7.1 权限类型
**功能描述**:  
应用需要的各项系统权限

**权限清单**:

| 权限类型 | 用途 | 请求时机 | 保留策略 |
|---------|------|----------|----------|
| 通知权限 | 接收消息通知 | 首次登录 | 一次授权永久保留 |
| 相机权限 | 拍照设置头像 | 点击拍照时 | 一次授权永久保留 |
| 相册权限 | 选择照片 | 点击相册时 | 一次授权永久保留 |
| 麦克风权限 | 语音通话 | 发起通话时 | 一次授权永久保留 |
| 位置权限 | 匹配附近的人 | 每次匹配时 | 每次使用都确认 |

#### 3.7.2 权限请求流程
**功能描述**:  
规范的权限请求流程

**流程设计**:
```
触发功能 → 检测权限状态
  ↓
未授权 → 显示权限说明弹窗
  ↓
用户点击"去开启" → 系统权限弹窗
  ↓
授权成功 → 执行功能
  ↓
授权失败 → 提示"需要权限才能使用"
  ↓
永久拒绝 → 显示"去设置"弹窗 → 跳转系统设置
```

#### 3.7.3 权限弹窗设计
**功能描述**:  
统一的权限请求弹窗

**弹窗元素**:
- 大图标（权限类型图标）
- 标题（如"开启相机权限"）
- 说明文案（如"开启相机权限后，你可以拍摄照片设置头像"）
- 按钮: "暂不开启" / "去开启"

**设计要求**:
- 底部弹出
- 半透明背景
- 圆角24px
- 极简设计，无边框

```

  ↓

匹配中（2.5秒）→ 匹配成功 → 弹出用户卡片

进入聊天页面 → 开始聊天

  ↓
30分钟后 → 可以添加好友
  ↓

```
启动APP → 启动页 → 自动登录
  ↓
主页面 → 查看消息列表
