# 瞬 FlashChat - 产品需求文档 (PRD)

**版本**：V1.0.0  
**文档日期**：2026年2月23日  
**产品名称**：瞬聊 FlashChat  
**产品定位**：24小时限时匿名社交应用

---

## 一、产品概述

### 1.1 产品愿景
打造一个纯粹、克制的匿名社交平台，让用户在24小时的限定时间内，与陌生人建立真实而短暂的连接。

### 1.2 核心理念
- **限时性**：24小时后对话自动消失，珍惜当下
- **匿名性**：渐进式身份解锁，保护隐私
- **克制性**：每日20次匹配机会，避免过度沉迷
- **纯粹性**：极简设计，专注对话本身

### 1.3 目标用户
- 年龄：18-35岁
- 特征：追求真实社交、注重隐私、喜欢极简设计
- 场景：深夜失眠、情感倾诉、随机聊天

## 二、功能需求

### 2.1 用户系统

#### 2.1.1 登录注册
**功能描述**：
- 手机号验证码登录
- 无需设置密码
- 首次登录自动注册

**交互流程**：
1. 用户输入11位手机号
2. 点击"发送验证码"（60秒倒计时）
3. 输入6位验证码
4. 验证成功后进入主页

**技术实现**：
- Mock验证码：123456（开发阶段）
- 真实环境：对接短信服务商API

#### 2.1.2 启动页
**功能描述**：
- 应用启动时展示品牌形象
- 光球呼吸动画 + 星空背景
- 持续2-3秒后自动跳转

**设计规范**：
- 背景：纯黑色 (#000000)
- 光球：白色渐变，呼吸效果
- 星空：随机闪烁的小光点

---

### 2.2 匹配系统

#### 2.2.1 匹配机制
**功能描述**：
- 每日20次匹配机会
- 每天早上9点重置次数
- 随机匹配在线用户

**匹配规则**：
- 优先匹配同城用户
- 避免重复匹配近期聊过的用户
- 匹配延迟2.5秒（营造期待感）

#### 2.2.2 次数显示
**功能描述**：
- 20个圆点阵列展示剩余次数
- 已用次数：白色高亮 + 光晕
- 未用次数：半透明灰色

**交互反馈**：
- 次数用完时，按钮变灰
- 显示"明天再来"提示

#### 2.2.3 招呼语系统
**功能描述**：
- 6个快捷招呼语：嗨、你好、在吗、聊聊、失眠了、晚安
- 自定义输入（最多25字）
- 快捷语和自定义二选一

**交互规则**：
- 选择快捷语后，自定义输入框禁用
- 输入自定义内容后，快捷语变灰
- 必须选择一种方式才能发送

#### 2.2.4 匹配结果
**功能描述**：
- 显示匹配用户的基本信息
- 头像：匿名占位符
- 昵称：显示"神秘人"
- 状态：用户设置的个性签名
- 距离：显示大致距离（如"3km"）

**操作选项**：
- **发送招呼**：消耗1次机会，进入聊天
- **换一个**：消耗1次机会，重新匹配

---

### 2.3 聊天系统

#### 2.3.1 对话机制
**功能描述**：
- 24小时限时对话
- 倒计时显示剩余时间
- 到期后对话自动消失

**时间规则**：
- 从发送第一条招呼开始计时
- 倒计时格式：HH:MM:SS
- 剩余1小时时，倒计时变红色提醒

#### 2.3.2 渐进解锁
**功能描述**：
- 初始状态：双方完全匿名
- 15分钟后：解锁头像和昵称
- 30分钟后：解锁个人主页（查看状态、兴趣标签）

**解锁提示**：
- 顶部显示解锁进度条
- 解锁时弹出提示："已解锁对方头像"

#### 2.3.3 消息功能
**功能描述**：
- 文字消息（最多300字）
- 消息气泡设计：深灰色，不对称圆角
- 自动滚动到最新消息

**交互细节**：
- 输入框：多行文本，自动扩展
- 发送按钮：输入内容后激活（白色）
- 消息状态：发送中、已送达

#### 2.3.4 Mock自动回复
**功能描述**（开发阶段）：
- 用户发送消息后2秒自动回复
- 随机回复内容库：
  - "哈哈哈"
  - "是吗"
  - "然后呢"
  - "有意思"
  - "我也这么觉得"

---

### 2.4 消息列表

#### 2.4.1 对话列表
**功能描述**：
- 显示所有进行中的对话
- 按最后消息时间排序
- 显示未读消息数量

**列表项信息**：
- 头像：根据解锁状态显示
- 昵称：神秘人 / 真实昵称
- 最后消息：预览前20字
- 时间：刚刚 / X分钟前 / X小时前
- 倒计时：剩余X小时X分钟

#### 2.4.2 未读提示
**功能描述**：
- 红点 + 数字显示未读数
- 最多显示99+
- 进入对话后自动清除

#### 2.4.3 滑动删除
**功能描述**：
- 左滑显示删除按钮
- 删除后对话永久消失
- 删除前二次确认

---

### 2.5 个人中心

#### 2.5.1 用户信息
**功能描述**：
- 显示头像（可更换）
- 显示手机号（脱敏）
- 显示个性签名（可编辑）

#### 2.5.2 设置项
**功能描述**：
- 关于瞬聊：版本号、产品介绍
- 隐私政策：查看隐私条款
- 用户协议：查看使用协议
- 退出登录：二次确认

## 三、设计规范

### 3.1 视觉风格

#### 3.1.1 色彩系统
```
主色调：
- 纯黑：#000000（背景）
- 纯白：#FFFFFF（主文字、按钮）

辅助色：
- 深海蓝：#0A0E27（卡片背景）
- 浅灰：#B8B8B8（次要文字）
- 深灰：#6B6B6B（三级文字）
- 错误红：#FF4444（警告、倒计时）

透明度变体：
- white05: rgba(255,255,255,0.05)
- white08: rgba(255,255,255,0.08)
- white12: rgba(255,255,255,0.12)
- white20: rgba(255,255,255,0.20)
```

#### 3.1.2 字体系统
```
字号：
- 标题：24px
- 大标题：20px
- 正文：16px
- 辅助文字：14px
- 小字：12px

字重：
- 超轻：300（主要使用）
- 常规：400
- 中等：500

字间距：
- 标题：2px
- 正文：1px
- 小字：0.5px
```

#### 3.1.3 圆角规范
```
- 按钮：50px（完全圆角）
- 卡片：24px
- 小卡片：12px
- 消息气泡：18px/4px（不对称）
```

### 3.2 动画规范

#### 3.2.1 时长
```
- 快速：200ms（按钮点击）
- 标准：300ms（页面切换）
- 慢速：3000ms（光球呼吸）
```

#### 3.2.2 缓动函数
```
- 进入：ease-out
- 退出：ease-in
- 循环：ease-in-out
```

### 4.3 数据模型

#### 4.3.1 用户模型
```dart
class User {
  String id;              // 用户ID
  String phone;           // 手机号
  String nickname;        // 昵称
  String avatar;          // 头像URL
  String status;          // 个性签名
  String distance;        // 距离
  DateTime createdAt;     // 注册时间
}
```

#### 4.3.2 消息模型
```dart
class Message {
  String id;              // 消息ID
  String threadId;        // 对话ID
  String content;         // 消息内容
  bool isMe;              // 是否是我发送的
  DateTime timestamp;     // 发送时间
}
```

#### 4.3.3 对话模型
```dart
class ChatThread {
  String id;              // 对话ID
  User otherUser;         // 对方用户
  int unlockLevel;        // 解锁等级（0/1/2）
  int unreadCount;        // 未读数量
  DateTime createdAt;     // 创建时间
  DateTime expiresAt;     // 过期时间
}
```
