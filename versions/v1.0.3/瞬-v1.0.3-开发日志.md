## 一、Flutter 前端优化

### 1.1 启动页淡出过渡（splash_screen.dart）

**问题**：导航前直接 `context.go()` 造成硬切换，视觉跳跃感强。
**优化**：导航前先调用 `_controller.reverse()` 执行淡出动画，过渡自然柔和。

```dart
// 优化前
context.go(authProvider.isLoggedIn ? '/main' : '/login');

// 优化后
await _controller.reverse();
if (!mounted) return;
context.go(authProvider.isLoggedIn ? '/main' : '/login');
```

---

### 1.2 底部导航栏动画修复（main_screen.dart）

**问题**：`TweenAnimationBuilder` 的 `begin` 和 `end` 均为 `isSelected ? 1.0 : 0.0`，值完全相同，动画从不执行。
**修复**：将 `begin` 固定为 `0.0`，`end` 动态绑定选中状态，Tab 切换时可见平滑过渡。

```dart
// 修复前
tween: Tween(begin: isSelected ? 1.0 : 0.0, end: isSelected ? 1.0 : 0.0),

// 修复后
tween: Tween(begin: 0.0, end: isSelected ? 1.0 : 0.0),
```

---

### 1.3 举报功能接入真实 API（report_screen.dart）

**问题**：举报提交使用 `Future.delayed` 模拟，实际不发送任何请求。
**优化**：接入 `ApiClient.instance.post('/report', data: {...})`，带完整错误处理链路。

- 成功：Toast 提示「举报已发送」并 pop 返回
- `ApiException`：显示服务端错误描述
- 其他异常：显示通用失败提示

---

### 1.4 消息列表底部弹窗动画统一（messages_tab.dart）

**问题**：长按会话弹窗使用硬编码 `BorderRadius.vertical(top: Radius.circular(16))` 与全局设计语言不一致。
**优化**：改用 `AppOverlay.sheetAnimationStyle` + `AppOverlay.sheetBorderRadius`，与其他底部弹窗统一。

---

### 1.5 匹配成功卡片入场动画（match_tab.dart）

**问题**：匹配成功后卡片直接出现，无过渡感。
**优化**：用 `AnimatedSwitcher` + `FadeTransition` + `SlideTransition` 组合实现 320ms easeOutCubic 入场：
- 从下方 4% 偏移滑入 + 淡入
- 退出时淡出

## 二、后端安全加固

### 2.1 OTP 发送频率限制（auth.service.ts）

```
同手机号 60 秒内只能请求一次
超限返回 HTTP 429，含剩余等待秒数
```

### 2.2 OTP 暴力破解保护（auth.service.ts）

```
连续验证失败 5 次 → 锁定该手机号 10 分钟
清除 OTP session，后续请求直接拦截
HTTP 429 + 中文说明
```

### 2.3 举报去重（report.service.ts）

```
同用户对同一目标 24 小时内重复举报静默返回 dedup
```

### 2.4 举报每小时频率上限（report.service.ts）

```
每用户每小时最多提交 10 条举报
超限静默返回 rate_limited（滑动时间窗口）
```

### 2.5 生产环境强制校验（app.setup.ts）

```
生产环境强制校验 JWT_SECRET（≥32字符）和 DATABASE_URL
CORS 生产环境限制 ALLOWED_ORIGINS 白名单
ValidationPipe 全局启用
```

## 五、P0 真机测试准备（补充）

### 5.1 Helmet 安全响应头（app.setup.ts）

安装并启用 `helmet` 中间件，一次性覆盖：
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Strict-Transport-Security`
- `X-XSS-Protection`
- `Referrer-Policy`

`crossOriginResourcePolicy: { policy: 'cross-origin' }` 允许 `/media/` 静态资源跨域加载。

### 5.2 账号注销全链路

**后端**：
- `auth.service.ts` 新增 `deleteAccount(actor, token)`：软删除用户（phone 改为 `deleted:{userId}`）、清除所有 session
- `auth.controller.ts` 新增 `DELETE /auth/account`，`@UseGuards(AuthGuard)` 保护

**Flutter**：
- `auth_service.dart` 新增 `deleteAccount()`：调用 `DELETE /auth/account`，清除本地 session
- `auth_provider.dart` 新增 `deleteAccount()` 方法：调用 service、清空所有本地状态、触发登出流程
- `settings_screen.dart` 退出登录按钮下方新增「注销账号」TextButton，点击弹出二次确认 Dialog，确认后执行注销并跳转登录页
