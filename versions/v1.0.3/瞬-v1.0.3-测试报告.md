## 当前失败项

本地执行 `flutter test` 仍失败，主要问题如下：

### 1. 插件依赖未 mock

- `permission_handler` 在测试环境抛出 `MissingPluginException`
- 直接影响：
  - `provider_performance_test`
  - `auth_provider_test`

### 2. 设置模块测试与现代码不一致

- `settings_provider_test.dart` 仍在访问已删除的：
  - `dayThemeEnabled`
  - `updateDayThemeEnabled`
- 当前 `SettingsProvider` 已固定夜间主题，不再提供这组接口

### 3. Smoke test 包装不完整

- `main_screen_smoke_test.dart` 缺少：
  - `NotificationCenterProvider`
  - `GoRouter`
- 结果导致：
  - `ProviderNotFoundException`
  - `No GoRouter found in context`

### 4. 个别断言与现逻辑偏离

- `profile_provider_test.dart` 中关于主页透明展示条件的断言已与当前实现不一致

## 测试结论

`v1.0.3` 具备基础自动化测试框架，但当前还不能视为“测试基线稳定”。

当前更适合的判断是：

- 代码主链路已基本成型
- 自动化测试还需要一轮修复和收口
- 下一阶段应先恢复 `flutter test` 全绿，再继续版本迭代
