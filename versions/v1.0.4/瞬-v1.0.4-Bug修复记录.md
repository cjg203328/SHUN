# 瞬 v1.0.4 Bug修复记录

## 当前已处理

### 测试基线问题

- 修复测试环境缺少 `permission_handler` 插件实现的问题
- 修复 smoke test 缺少 `NotificationCenterProvider` 和 `GoRouter` 上下文的问题
- 清理 `SettingsProvider` 已删除接口对应的过期测试

### 状态一致性问题

- 修复关闭竖版全屏背景后，透明主页状态未同步关闭的问题
- 修复消息撤回仅在 UI 层限制、Provider 层未校验的问题，当前统一限制为发送后 2 分钟内可撤回
- 修复 WebSocket `msg.new` 重复到达时会重复增加未读数的问题，改为按消息 ID 幂等处理
- 修复对端已读事件会把整段会话全部标记为已读的问题，改为按 `lastReadMessageId` 精确回执
- 修复远端消息回补时会覆盖本地图片预览的问题，当前相同图片消息会优先保留本地可恢复信息
- 修复远端历史延迟返回时会刷掉本地已发送消息的问题，当前对近期已发送消息保留同步保护窗口
- 修复失败图片消息点击重试会被当成文本 `[图片]` 重发的问题，当前仅允许失败文本消息重试
- 修复本地失败/发送中消息与远端成功消息回补时可能双份并存的问题，当前会按内容类型与时间窗口进行归并
- 修复自己发送的消息被 `msg.new` 回推时会按“新来消息”处理的问题，当前会优先归并到本地发送结果，不再增加未读和通知
- 修复自己发送的消息经 `msg.new` 自回环成功时不会补亲密度的问题，当前已与 `ACK` 成功链路保持一致
- 修复连续发送图片时远端回补可能按 `[图片]` 文案误归并到错误本地图片的问题，当前图片归并会额外校验阅后即焚和质量指纹
- 修复会话已读后通知中心同线程聊天通知仍保持未读的问题，当前线程已读会同步清理对应聊天通知未读状态
- 修复同一条聊天消息可能在通知中心重复入库的问题，当前聊天通知按 `threadId + messageId` 幂等写入

### 可维护性优化

- 将 `ChatProvider` 按职责拆分为存储、线程、消息、实时事件 4 个 `part` 文件，降低后续聊天功能扩展成本

## 继续跟进

- 收口聊天离线补偿与已读同步边界
- 为图片消息、会话恢复和通知联动补更完整的回归覆盖
