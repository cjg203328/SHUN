# 瞬 App 区域测试部署说明

这份文档按“零基础也能照着做”的方式写，目标是先把 **后端跑起来 + 内网穿透 + 真机安装测试包**。

## 一、你现在已经可以直接交付的内容

- Flutter App 主体功能已经可用：登录、资料、设置、匹配、好友、聊天、图片上传、通知中心。
- 后端接口已经接通，聊天 WebSocket 也已接通。
- 设置页里的 `关于瞬`、`隐私政策`、`用户协议` 已经改成正式页面，不再是占位提示。
- 根目录已有可打包脚本：`build.bat`
- 我新增了两个更适合你现在使用的脚本：
  - `start-test-backend.bat`：一键启动测试后端
  - `build-test-apk.bat`：一键把内网穿透地址写进 App 并打 APK

## 二、你要准备什么

### 1）本机需要有

- Node.js
- npm
- Flutter
- 一个可用的内网穿透工具

### 2）内网穿透要求

- **必须支持 HTTPS**
- **必须支持 WebSocket**
- 需要把你电脑本地的 `3000` 端口映射出去

如果你的穿透地址是：

`https://abc.example.com`

那么 App 实际会使用：

- API 地址：`https://abc.example.com/api/v1`
- 媒体地址：`https://abc.example.com/media`

## 三、第一步：启动后端

### 最简单方式

双击运行：

`start-test-backend.bat`

它会自动按“区域测试最省事”的方式启动：

- `PORT=3000`
- `USER_STORE_DRIVER=memory`
- `AUTH_RUNTIME_DRIVER=memory`
- `RUNTIME_STATE_DRIVER=memory`

### 如果脚本提示没安装依赖

先执行：

```bat
cd backend\server
npm install
```

然后再双击 `start-test-backend.bat`

### 启动成功后你可以检查

浏览器打开：

`http://127.0.0.1:3000/api/docs`

只要这个页面能打开，说明后端基本已经起来了。

## 四、第二步：做内网穿透

把你电脑本地的 `3000` 端口暴露出去。

你最终会得到一个 **HTTPS 公网地址**，例如：

`https://abc.example.com`

### 这里一定要注意

- 不能只映射 `api`，要让整个站点可访问，因为图片走 `/media`
- 不能只支持 HTTP，真机安装包建议一定走 HTTPS
- 不能不支持 WebSocket，否则聊天实时消息会异常

## 五、第三步：打区域测试 APK

双击运行：

`build-test-apk.bat`

然后输入你的内网穿透 HTTPS 地址，例如：

`https://abc.example.com`

它会自动生成：

- `SUNLIAO_API_BASE_URL=https://abc.example.com/api/v1`
- `SUNLIAO_MEDIA_BASE_URL=https://abc.example.com/media`

并调用根目录 `build.bat` 生成测试包。

### 打包完成后的 APK 位置

`apk-output\shun-v1.0.3-release.apk`

## 六、第四步：安装到手机测试

安装 APK 后，建议按下面顺序测试：

1. 打开 App
2. 输入手机号
3. 发送验证码
4. 输入验证码 `123456`
5. 登录后进入首页
6. 依次测试资料、设置、匹配、聊天、好友、通知

## 七、区域测试阶段的重要说明

### 1）当前验证码

当前测试后端验证码固定为：

`123456`

这只是为了让你先跑通测试流程，**正式上线前必须换成真实短信服务**。

### 2）当前数据存储方式

当前 `start-test-backend.bat` 使用的是 `memory` 存储。

意思是：

- 你关闭后端进程后，测试数据会丢失
- 适合你现在做区域测试
- 不适合正式上线

### 3）当前发布包签名

当前项目保留了 debug 签名兜底能力，便于区域测试打包。

正式上架前必须改成你自己的正式签名文件。

## 八、常见问题

### 1）手机能打开 App，但登录/聊天失败

优先检查：

- 后端是否还在运行
- 内网穿透地址是否失效
- 穿透是否支持 WebSocket
- 你是否用了 HTTPS 地址

### 2）图片发不出去或看不到

优先检查：

- 穿透地址是否能访问 `/media`
- 打包时是否正确写入了 `SUNLIAO_MEDIA_BASE_URL`

### 3）换了新的穿透地址后 App 还是连旧地址

需要 **重新打包一次 APK**，因为接口地址是在打包时写进去的。

## 九、我建议你现在的实际执行顺序

按这个顺序最稳：

1. 先运行 `start-test-backend.bat`
2. 确认 `http://127.0.0.1:3000/api/docs` 可打开
3. 再做内网穿透，拿到 HTTPS 地址
4. 运行 `build-test-apk.bat`
5. 安装 `apk-output\shun-v1.0.3-release.apk`
6. 真机测试完整流程
